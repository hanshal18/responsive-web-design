DOCTYPE element

The <!DOCTYPE html> declaration is an important part of HTML documents, signaling to the web browser that the document is written in HTML5. It should be the very first line in an HTML file, preceding the <html> tag.

Key Points about <!DOCTYPE html>:

  • Purpose: It tells the browser to render the page in standards mode, ensuring consistent behavior across different browsers.

  • HTML5 Standard: The declaration is simple and does not require a reference to a DTD (Document Type Definition), as was the case in earlier HTML versions. This simplification reflects the move towards a more flexible web.

  • No Closing Tag: The <!DOCTYPE html> declaration is not a tag itself and does not require a closing tag.

  • Compatibility: Using this declaration helps avoid quirks mode, where browsers may render the page differently based on outdated standards.

HTML element

HTML (HyperText Markup Language) tags are the building blocks of web pages. They are used to structure and format content on the internet. Each tag is enclosed in angle brackets, like <tagname> , and usually comes in pairs, with an opening tag and a closing tag (e.g., <p> for opening and <p> for closing).

Key Points about HTML Tags:

  • Structure: Tags define elements like headings, paragraphs, links, images, and lists.

  • Attributes: Tags can include attributes to provide additional information, such as href for links or src for images (e.g., <img src="image.jpg" alt="description">).

  • Nested Tags: Tags can be nested within one another to create complex structures (e.g., <div> <p>Text</p> </div>).

  • Tags: HTML5 introduced semantic tags (e.g., <header>, <footer>, <article>, <section>) that provide meaning to the structure and help with SEO and accessibility.

  • Self-Closing Tags: Some tags do not require a closing tag (e.g., <br> for a line break or <img> for images).

HEAD element

The <head> element is a crucial part of an HTML document, positioned between the <!DOCTYPE html> declaration and the <body> tag. It contains metadata and links to resources that provide information about the document but are not displayed directly on the webpage.

BODY element

The <body> element is a fundamental part of an HTML document that contains the content intended for display in a web browser. Everything that users see on a webpage—text, images, links, videos, and interactive elements—resides within this tag.

FOOTER element

The <footer> element in HTML is a semantic tag used to define the footer section of a webpage or a specific section within a document. It typically contains information related to the content it accompanies, such as copyright details, author information, links to related documents, or contact information.
